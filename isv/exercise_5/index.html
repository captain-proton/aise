Erläuterungen zur Lösung

(a) All used classes are displayed on a circle. With a mouseover the full class name is shown on the upper left corner of the diagram.

(b) By default all link types are selected and shown in the diagram. After (de-)selection of any option and submission, the diagram is rendered again. Each cluster is divided with additional whitespace between the class names. If no link type is selected no clustering can be done, due to the cause that each vector is a null vector. A warning message is displayed in this case.

(c) Each connection of two classes is rendered as a path inside the circle. On mouseover outgoing connections (source to target) are highlighted with a specific color. The source node is displayed in bold font. The target node is displayed in the color of the link type. There are two types of cluster building mechanisms available. The first is defined by the task. The second builds the vectors, that are used by the kmeans clustering, with the count of how many connections the class has in one type. Look at the description of the function buildLinkTypeCountNodeVectors for more information.
